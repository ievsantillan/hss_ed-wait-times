# HSS Emergency Department Wait Times

HSS (Health Shared Services) Emergency Department Wait Times is a public, live emergency department wait-times app for Alberta. It refreshes every 2 minutes from the Alberta Health Services feed, uses HSS branding, and is built on Rayfin with Microsoft Fabric.

## Disclaimer

Wait times are approximate and provided for information only. They are not medical advice and may change without notice. In a life-threatening emergency, call 911. Wait-time data is sourced from Alberta Health Services.

## Features

- Live 2-minute polling from the Alberta Health Services wait-times feed.
- Coverage for all 7 AHS regions: Calgary, Edmonton, Red Deer, Lethbridge, Medicine Hat, Grande Prairie, and Fort McMurray.
- Search, region, and facility-type filters.
- Nearest-to-me sorting using browser geolocation.
- Favourite facilities stored on the user's device.
- Status flags for closed, closing-soon, and unavailable facilities.
- Daily local news aggregation.
- Admin CRUD (Fabric sign-in required) for facility records, overrides, and non-AHS facilities.
- WCAG 2.1 AA target with keyboard navigation and reduced-motion support.
- PWA manifest for installable app experiences.

## Architecture

See [docs/architecture.md](docs/architecture.md) for the full architecture diagram and data-flow notes.

At a high level, the browser polls the Alberta Health Services wait-times API directly every 2 minutes, with no backend in the request path. News and trend history are served as static JSON files (`public/news.json`, `public/wait-history.json`) regenerated by scheduled GitHub Actions. A daily news job aggregates local news per region, and an hourly snapshot job appends wait readings for trends. Only the staff admin area talks to the Rayfin Data API, and it requires Fabric sign-in.

## Data model summary

The patient-facing experience reads only static files and the live AHS feed (no backend). The Rayfin Data API is used exclusively by the authenticated admin area:

- `Facility`: facility catalog, region, category, location, contact details, active status, and admin override fields (`authenticated` role).
- `WaitReading`: admin-entered wait readings and overrides (`authenticated` role).

News (`news.json`) and trend history (`wait-history.json`) are static files committed by the scheduled jobs, not database entities.

## Access model

The patient-facing app has no login and no backend dependency: live wait times come straight from the AHS feed, and news/trends come from static JSON. Public users never authenticate.

The Rayfin Data API rejects anonymous data access at deploy time on Microsoft Fabric, so all database access is gated behind Fabric sign-in. Only staff use the `/admin` area, which prompts for Fabric SSO before reading or writing `Facility` and `WaitReading` records.

## Getting started

### Prerequisites

- Node.js 24.
- Docker, required for a local Rayfin backend.
- Access to a Rayfin and Microsoft Fabric environment for deployment.

### Install dependencies

```powershell
npm install
```

### Environment variables

Local development needs:

```powershell
$env:VITE_RAYFIN_API_URL = "https://your-rayfin-api"
$env:VITE_RAYFIN_PUBLISHABLE_KEY = "pk-your-publishable-key"
```

Deployed Fabric builds also use:

```powershell
$env:VITE_FABRIC_WORKSPACE_ID = "your-workspace-id"
$env:VITE_FABRIC_ITEM_ID = "your-fabric-item-id"
$env:VITE_FABRIC_PORTAL_URL = "https://app.fabric.microsoft.com"
```

### Run locally

```powershell
npm run dev
```

Open http://localhost:5173.

## Scripts

| Command | Description |
| --- | --- |
| `npm run dev` | Starts the Rayfin-backed Vite development experience. |
| `npm run build` | Type-checks and builds the production static app. |
| `npm run build:fabric` | Builds the static app for Rayfin Fabric static hosting. |
| `npm run lint` | Runs ESLint. |
| `npm run test` | Runs the Vitest test suite. |
| `npm run seed` | Bootstraps the Rayfin facility catalog (authenticated). |
| `npm run news` | Regenerates `public/news.json` from regional news sources. |
| `npm run snapshot` | Appends an hourly point to `public/wait-history.json`. |

Seed runs against the authenticated Rayfin backend; set `RAYFIN_API_URL` and `RAYFIN_PUBLISHABLE_KEY` plus admin credentials before running it. News and snapshot scripts need no backend access: they write `public/news.json` and `public/wait-history.json` directly. Daily news and hourly snapshot jobs run through GitHub Actions in `.github/workflows` and commit the regenerated JSON.

## Deployment

Sign in and deploy with Rayfin:

```powershell
npx rayfin login
npx rayfin up
npx rayfin up status
```

`npx rayfin up` builds the static app, deploys it, and applies schema migrations.

Scheduled GitHub workflows regenerate static JSON and commit it back to the repository. They require no Rayfin secrets; the workflows use the built-in `GITHUB_TOKEN` with `contents: write` permission to push the updated `public/news.json` and `public/wait-history.json`.

After deployment, regenerate PWA icons from `src/assets/hss-logo.jpg` and place them at `public/icons/icon-192.png` and `public/icons/icon-512.png`. Update the placeholder sitemap host in `public/sitemap.xml` from `https://hss-ed-wait-times.example` to the deployed host.
